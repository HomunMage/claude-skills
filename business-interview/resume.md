# Resume — CV & Portfolio Preparation

> **Cross-reference**: Also read [interview.md](interview.md) — your resume sets up the stories you'll tell in interviews.

## Resume Philosophy

- A resume is a **marketing document**, not a biography
- Goal: get the interview, not tell your whole life story
- Tailor for each application — one generic resume for all is ineffective
- Recruiter spends ~6 seconds on first scan — front-load impact

## Structure

### Header
- Name, contact (email, phone), location (city only), LinkedIn, portfolio link
- No photo (unless culturally required), no age, no marital status

### Summary (optional, 2-3 lines)
- Who you are + key strength + what you bring
- Only include if it adds value beyond the job title

### Experience (most important section)
- Reverse chronological
- Each entry: Company, Title, Duration
- Bullet points format: **Action verb + what you did + measurable result**
  - "Reduced build time by 40% by implementing parallel CI pipeline"
  - "Led a team of 5 to ship MVP in 3 months, acquired 1,000 beta users"
  - NOT: "Responsible for managing the team" (passive, no result)
- 3-5 bullets per role, more for recent/relevant roles
- Quantify everything possible: %, $, time saved, users, scale

### Skills
- Separate: technical skills, tools, languages, frameworks
- Only list skills you can defend in an interview
- Match keywords from job description (ATS optimization)

### Education
- Degree, school, year
- Only include GPA if exceptional; omit after 2+ years of experience

### Projects / Portfolio (for creative & technical roles)
- Link to live work, GitHub, portfolio site
- Brief description + your specific contribution + tech stack
- Quality > quantity — 3-5 best projects

## Tailoring Strategy

### Per Application
1. Re-read the job description
2. Identify the top 3-5 requirements
3. Reorder bullets to lead with most relevant experience
4. Mirror their language (if they say "cross-functional", use that term)

### ATS (Applicant Tracking System) Tips
- Use standard section headers (Experience, Education, Skills)
- No tables, columns, or complex formatting — ATS can't parse them
- Include keywords from JD naturally in context
- Submit as PDF unless they specify otherwise

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Listing duties instead of achievements | Use "accomplished X by doing Y, resulting in Z" |
| Too long (3+ pages) | 1 page for < 10 years experience, 2 max |
| Generic objective statement | Replace with tailored summary or remove |
| Gaps unexplained | Brief note: freelance, study, personal project |
| Typos / inconsistent formatting | Proofread 3x, ask someone else to review |
| Including everything ever done | Curate — only relevant experience |

## Portfolio Strategy (creative / technical)

- **Show process, not just results** — sketches, iterations, decisions
- **Case studies** > gallery — explain the problem, your approach, the outcome
- **Tailor showcase** to target role — game studio? Show game work first
- **Keep it updated** — remove old/weak pieces, add recent best work
- **Make it accessible** — fast loading, mobile-friendly, clear navigation
